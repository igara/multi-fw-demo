# demo.code-workspace

このファイルは、VS Codeの**マルチルートワークスペース**設定ファイルです。モノレポ構成のプロジェクトにおいて、複数のパッケージを個別のルートフォルダとして管理できるようにしています。

## 構成内容

```jsonc
{
  "folders": [
    {
      "path": "./",
      "name": "multi-fw-demo"
    },
    {
      "path": "packages/shadcn"
    },
    {
      "path": "packages/applications/nextjs"
    }
  ]
}
```

## フォルダ構成

### 1. multi-fw-demo (`./`)
- プロジェクトのルートディレクトリ
- 全体的な設定ファイル（`package.json`, `pnpm-workspace.yaml`など）を含む
- モノレポ全体の管理を担当

### 2. packages/shadcn
- shadcn/uiコンポーネントライブラリのパッケージ
- 再利用可能なUIコンポーネントを管理
- Storybookによるコンポーネントカタログを提供

### 3. packages/applications/nextjs
- Next.jsアプリケーションのパッケージ
- shadcnパッケージを利用するフロントエンドアプリケーション
- 実際のアプリケーション実装を担当

## マルチルートワークスペースのメリット

### 1. パッケージごとの独立した設定
各パッケージが独立したルートとして扱われるため、パッケージ固有の設定が正しく機能します:
- ESLint設定
- TypeScript設定（`tsconfig.json`）
- フォーマッター設定（Biome、Prettierなど）

### 2. 効率的なファイル管理
- 各パッケージのファイル検索やナビゲーションが容易
- パッケージごとにスコープを絞った検索が可能
- ファイルツリーが整理され、必要なパッケージに素早くアクセス可能

### 3. 柔軟な開発環境
- パッケージごとに異なる拡張機能の設定が可能
- パッケージ固有のタスクやデバッグ設定を分離
- チームメンバーが特定のパッケージに集中しやすい

## 使用方法

VS Codeで`demo.code-workspace`ファイルを開くと、定義されたすべてのフォルダがワークスペースに読み込まれます:

```bash
code demo.code-workspace
```

または、VS Code内で「ファイル」→「ワークスペースをファイルから開く」を選択してこのファイルを開きます。
